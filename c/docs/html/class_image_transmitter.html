int ImageTransmitter::streamImage ( Mat *  image)

This method will stream via udp the image to the remote device.

Parameters
imageThis is the image that is to be sent.
Returns
The return will be 0 if successful or -1 if there is a failure.

1.0 If the image and destination machine are not null,

1.1 Increment the image count.

1.2 Initialize the socket sockfd to be a DGRAM.

1.3 If there is an error with 1b, abort with an error message and return -1.

1.4 Get the host by name and set up a server instance.

1.5 If the server is NULL, print out an error and return -1.

1.6 Set up the rest of the UDP parameters.

1.7 Set the port for the system.

1.8 Obtain the image rows, columns, message size, and required buffer allocation size (which is ((3 * columns + 24) * linesPerUDPDatagram) + 4).

1.9 Allocate a buffer of message size length on the heap.

1.10 Obtain the current timestamp in ms using the time_util library.

1.11 Declare a pointer to a 32 bit integer that has the same address as the start of the buffer. The, set the first 32 bits of the buffer to be the network copnverted endianess of the linesPerUDPDatagram variable.

1.12 Declare a variable that will keep track of the index into the image (i.e. which row is being packed right now), as well as another variable which counts how many rows have been packed into the given UDP datagram.

1.11 Iterate (using index) so longas the index is less than rows in the image.

1.11.1 Using a for loop, place up to linesPerUDPDatagram within the UDP message. However, also make certain that we do not pack rows which do not exist into the UDP datagram. To do this, in addition to the udpLineNumber being less than linesPerUDPDatagram, the conditional on the for loop should also check to make sure that rowsPacked < rows (i.e. the number of rows that has been packed for transmission is less than the number of rows overall.

1.11.1.1 Starting after the initial 4 bytes of the udp, which contains the linesPerUDPDatagram attribute, create the portion of the message which has the following: Integer 0: The start time for the transmission Integer 1: The current timestamp for the current portion of the image Integer 2: The count of the image. Integer 3: The number of rows in the image. Integer 4: The number of columns in the image Integer 5: The index being sent Followed by: An array of columns * 3 bytes, representing the pixels in the current row.

The integers all need to have their endianess corrected before being sent. The data array does not.

Overall, there will be linesPerUDPDatagram instances of this in the allocated space, representing linesPerUDPDatagram lines being sent out at a time.

1.11.1.2 Increment the rows packed variable to indicate that another row has been packed.

1.11.2 Send to message to the destination.

1.11.3 Increment the index to account the lines that were sent.

1.12 Free the message that was allocated on the heap.

1.13 Close the socket down.
